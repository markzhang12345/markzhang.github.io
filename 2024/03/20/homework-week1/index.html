安装 pnpm

pnpm 是一个高效的包管理器，它提供了多种相较于 npm 的优点：

  1. 磁盘空间效率：pnpm 使用一种称为内容寻址文件系统的机制来存储依赖。它允许不同项目共享同一个版本的包，而不需要在每个项目中重复下载，从而节约了磁盘空间。

  2. 性能提升：pnpm 在执行安装时通常比 npm 更快，因为它避免了包的重复下载以及提高了包的链接速度。

  3. 严格的依赖管理：pnpm 创建一个非平坦的 node_modules 目录，确保了代码只能访问在 package.json 中声明的依赖项，减少了意外引用未声明的依赖的可能性。

  4. 原子式安装：如果安装过程被中断或失败，pnpm 不会在 node_modules 目录中留下无效状态，它保证要么全部完成要么什么都不做。

  5. **更好的支持 monorepo**：pnpm 支持 Workspace，能够有效地处理多包（多项目）存储库的依赖关系，这使得管理大型项目或多个相关项目变得更加方便。

  6. 干净的 Git diff：由于 pnpm 使用一个单独的文件来存储链接到包的信息（pnpm-lock.yaml），这样可以减少 Git diff 的大小，并且更容易在版本控制中理解依赖变更。

  7. 更好的下载量统计：pnpm 默认加了一个代理服务器来下载 npm 包。这样可以统计每天的下载量，即使是当您使用了私有包或其他源的包。

安装 Vite

Vite（法语中“快速”的意思）是一个现代化的前端构建工具，由 Vue.js 创建者尤雨溪开发。它旨在提供更快的开发服务器启动和更快的模块热替换（HMR）。与传统的前端构建工具（如 Webpack）相比，Vite 在开发模式下不使用打包操作，这带来了显著的性能提升。

请注意，全局安装后的 vite 可用于创建新项目或在全局范围内使用 vite 的功能，但在项目开发中，通常推荐将 vite 安装为项目的开发依赖项，以保证不同项目之间的版本独立。在后续的沙龙中我们需要频繁地使用 vite 进行项目的配置等相关工作。

任务二：配置 Git 环境

任务三：寒假作业重构

请您根据您的实际情况，结合在本次沙龙与先前的寒假作业答辩中的收获，重构您的寒假作业，并将寒假作业代码开源至 github 中。如果您希望获得更多反馈，可以将您的 github 项目链接以邮件的方式发送到 1185570212@mail.dlut.edu.cn。

对重构的具体分析可以参考 Martin Fowler 的 《重构: 改善既有代码设计》 一书，在 Martin Fowler 看来，重构是 “对软件内部结构的一种调整，目的是在不改变软件可观察行为的前提下，提高其可理解性，降低其修改成本。”

同时，重构具有以下的益处/必要性：

  1. 使添加新功能更加容易
  2. 使代码更为易懂
  3. 改进软件设计
  4. 寻找潜在 bug
  5. 提高开发效率
  6. 消除代码的异味
